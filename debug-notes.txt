Bug 1: "Error al cargar el documento" from Dashboard
- handleEditDocument in Dashboard.tsx line 327-345
- Does fetch(doc.fileUrl) where fileUrl is R2 public URL
- R2 bucket has NO CORS headers → browser blocks the fetch
- Solution: Add a server-side proxy endpoint /api/documents/proxy?url=... that fetches from R2 and returns the PDF

Bug 2: Word/docx conversion fails in production
- convertToPdf.ts uses LibreOffice headless for Office docs
- LibreOffice is NOT installed in Cloud Run production
- Solution: Use mammoth.js (already installed) to convert DOCX → HTML, then HTML → PDF with jsPDF (already installed) or pdf-lib
- For Excel/PPT: need alternative approach too
